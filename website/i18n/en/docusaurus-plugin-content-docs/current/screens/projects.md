# Projects Screen

## Purpose

The projects screen is the main catalog entry point for most users.

## Design

- list of projects;
- filter entry point;
- customer request flow where applicable;
- navigation into project detail and personal project flows.

## States

- initial loading;
- filtered or unfiltered result list;
- empty result set;
- backend error state.

## Requests

- project list loading;
- project filtering-related requests;
- optional project submission or request flows for customers.
