# Current Status

## Current scope

The mobile client already covers:

- GitHub-based sign-in and guest mode;
- projects catalog and project detail flows;
- ranking for projects and users;
- personal and project statistics;
- profile, settings, privacy, and feedback flows;
- Android and iOS targets from the same KMP codebase.

## Operational expectations

- local development depends on client config and the external Registry backend repository;
- release and CI/CD flows are part of the repository workflow;
- the docs site is the primary documentation surface for product, development, and operations.
